User Management

User management in Salesforce involves creating, maintaining, and controlling user accounts to ensure proper access and security within the organization's Salesforce instance.

1. Creating User Accounts

Administrators can create new user accounts to provide access to Salesforce for employees, partners, or customers. When creating user accounts, administrators specify various details such as username, email address, profile, role, and permissions.

2. Editing User Accounts

Admins can edit user account details as needed, such as updating contact information, changing roles or profiles, adjusting permissions, and modifying login settings.

3. Deactivating User Accounts

When an employee leaves the organization or no longer requires access to Salesforce, administrators can deactivate their user accounts to prevent further access. Deactivated users cannot log in to Salesforce but their data and records remain intact.

4. Profiles

Profiles in Salesforce define the permissions and settings that determine what users can do within the organization's Salesforce instance. Administrators assign profiles to user accounts to grant appropriate access and functionality based on roles and responsibilities.

5. Permission Sets

Permission sets allow administrators to grant additional permissions or access settings to specific users, beyond what is available in their assigned profiles. This enables more granular control over user access and functionality.

6. Roles

Roles in Salesforce define the hierarchy of data access within the organization. By assigning roles to user accounts, administrators control which records users can view, edit, or delete based on their position in the hierarchy.

7. Login Policies

Administrators can configure login policies to enforce security measures such as password requirements, login hours, IP restrictions, and multi-factor authentication to protect user accounts from unauthorized access.
